# My First Git & GitHub Guide

このガイドは、GitとGitHubを初めて使うためのステップバイステップガイドです。プロジェクトのセットアップから日々の開発サイクル、困ったときの対処法までを網羅します。

## 1. 初期設定とプロジェクトのセットアップ

### ステップ0: Gitのインストール

まだGitをインストールしていない場合は、公式サイトからダウンロードしてインストールします。

1.  **公式サイト:** [https://git-scm.com/](https://git-scm.com/)
2.  インストーラーをダウンロードし、基本的に全てデフォルト設定でインストールを完了します。
3.  ターミナル（コマンドプロンプトやPowerShell）で以下のコマンドを実行し、バージョンが表示されれば成功です。
    ```bash
    git --version
    ```

### ステップ1: プロジェクトフォルダに移動

ターミナルで、管理したいプロジェクトのフォルダに移動します。

```bash
# 例 (Windows): cd C:\Users\YourName\Documents\MyProject
# 例 (Mac): cd /Users/YourName/Documents/MyProject
cd [あなたのプロジェクトフォルダへのパス]
```

### ステップ2: Gitリポジトリの初期化

そのフォルダをGitの管理下に置きます。（プロジェクトごとに一度だけ実行）

```bash
git init
```
これで `.git` という隠しフォルダが作成され、変更履歴がここに保存されます。

### ステップ3: `.gitignore` ファイルの作成 (最重要！)

Gitに追跡させたくないファイル（自動生成ファイル、個人設定など）を指定する設定ファイルを作成します。

1.  プロジェクトのルートフォルダに `.gitignore` という名前のファイルを作成します。
2.  Pythonプロジェクト向けの一般的な内容を[こちら](https://github.com/github/gitignore/blob/main/Python.gitignore)などからコピーして貼り付けます。
3.  その他、プロジェクト固有で無視したいファイル/フォルダがあれば追記します。（例: `.serena/`）

> **なぜ重要？** `.gitignore` がないと、不要なファイルまでバージョン管理されてしまい、リポジトリが汚れたり、競合の原因になったりします。

### ステップ4: 最初のコミット（セーブ）

#### 4-1. 変更内容の確認

`git status` は、現在のプロジェクトの状態を確認するための最も重要なコマンドです。いつでも使いましょう。

```bash
git status
```

#### 4-2. ファイルをステージング

コミットに含めたいファイルを「ステージングエリア」という待機場所に追加します。

```bash
# 全ての変更をステージングする場合 (一番よく使う)
git add .

# 特定のファイルだけをステージングする場合
git add main.py .gitignore
```

#### 4-3. コミットの実行

ステージングした内容を、メッセージ付きでリポジトリに記録（セーブ）します。

```bash
git commit -m "Initial commit of the project"
```
-   `-m "..."` の部分は、**「何を変更したか」が分かりやすいメッセージ**にするのが鉄則です。
    -   良い例: `"ユーザー認証機能を追加"`, `"ヘッダーのバグを修正"`
    -   悪い例: `"修正"`, `"いろいろ変更"`

> **初回のみ:** ユーザー名とメールアドレスの設定を求められた場合は、画面の指示に従って設定してください。
>
> ```bash
> git config --global user.name "Your Name"
> git config --global user.email "your.email@example.com"
> ```

## 2. GitHubとの連携 (バックアップと共有)

ここまでの作業は自分のPC内（ローカルリポジトリ）だけのものです。次は、インターネット上の倉庫（リモートリポジトリ）であるGitHubにコードを保存します。

### ステップ1: GitHubにリポジトリを作成

1.  [GitHub](https://github.com/)にログインし、新しいリポジトリを作成します。
2.  リポジトリ名を決め、**Public** または **Private** を選択します。
3.  **`README`, `.gitignore`, `license` の追加チェックは全て外して**作成します。（ローカルで既に作成済みのため）

### ステップ2: ローカルとリモートを接続

リポジトリ作成後に表示されるコマンドをターミナルで実行します。

```bash
# 1. リモートリポジトリのURLを 'origin' という名前で登録
git remote add origin https://github.com/YourUsername/YourRepoName.git

# 2. 現在のブランチ名を 'main' にする (近年の推奨)
git branch -M main

# 3. ローカルの 'main' ブランチの内容をリモートの 'origin' にアップロード (push)
git push -u origin main
```
これで、ローカルのコミット履歴がGitHubにアップロードされます。GitHubのページを再読み込みして、ファイルが表示されていれば成功です。

## 3. 日々の開発サイクル

開発を進める際は、以下のサイクルを繰り返します。

1.  **コードを編集する**
    -   新機能の追加やバグ修正などを行います。

2.  **変更を確認する (`git status`)**
    -   どのファイルが変更されたかを確認します。

3.  **ステージングする (`git add`)**
    -   コミットしたい変更をステージングします。
        ```bash
        git add .
        ```

4.  **コミットする (`git commit`)**
    -   変更をローカルリポジトリに記録します。
        ```bash
        git commit -m "実装した内容の簡潔な説明"
        ```

5.  **プッシュする (`git push`)**
    -   ローカルのコミットをGitHubにアップロードしてバックアップします。
        ```bash
        git push
        ```

## 4. 困ったとき：過去の状態に戻す方法

### 状況1：コミット前の変更を全て取り消したい

-   **目的:** `git commit` する前に加えた変更を、全て直近のコミットの状態に戻す。
-   **コマンド:**
    ```bash
    git restore .
    ```
-   **特徴:** シンプルで安全。セーブせずにリセットする感覚。

### 状況2：過去の特定のコミットに完全に戻りたい

-   **目的:** いくつか前のコミット時点に、プロジェクト全体を完全に戻す。（途中のコミットは消える）
-   **手順:**
    1.  戻りたいコミットのIDを調べる。
        ```bash
        git log --oneline
        ```
    2.  指定したIDの状態に戻る。
        ```bash
        # 例: git reset --hard a1b2c3d
        git reset --hard <コミットID>
        ```
-   **特徴:** 強力だが、消えたコミットは元に戻せない。個人開発で「全てやり直したい」時に便利。

### 状況3：「特定のコミットだけ」を打ち消したい

-   **目的:** あるコミットで行った変更内容だけを、履歴を残したまま打ち消す。
-   **コマンド:**
    ```bash
    # 例: git revert a1b2c3d
    git revert <コミットID>
    ```
-   **特徴:** 打ち消し用の新しいコミットが作成されるため、履歴が改変されない。チーム開発でも安全に使える。

### Tips: よくある警告とエディタ

-   **改行コードの警告 (LF will be replaced by CRLF)**
    -   **意味:** Windows (CRLF) と Mac/Linux (LF) の改行コードの違いを、Gitが自動で統一してくれている親切なお知らせ。
    -   **対処:** 無視してOK。そのまま作業を続けて問題ありません。

-   **`git commit` したらVimエディタが開いた**
    -   **原因:** `git commit` コマンドに `-m "メッセージ"` を付け忘れたため。
    -   **対処 (保存して終了):** `Esc`キーを押し、`:wq` と入力して `Enter`。
    -   **対処 (中止して終了):** `Esc`キーを押し、`:q!` と入力して `Enter`。
    -   **予防:** コミット時は `-m` を付ける癖をつける。
