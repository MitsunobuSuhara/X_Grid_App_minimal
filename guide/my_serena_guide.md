### 【毎日の起動手順】Serenaを再び使い始めるとき

`Serena`は「サーバー」と「クライアント」の2つの部分で動いています。そのため、**PowerShellのウィンドウを2つ**使って、それぞれを起動する必要があります。

#### ステップ1: Serenaサーバーを起動する (1つ目のPowerShell)

まず、`Serena`の頭脳であるサーバーを起動します。

1.  **PowerShell**を開きます。
2.  プロジェクトフォルダに移動します。
    ```powershell
    cd C:\Users\mitsu\Desktop\X_Grid_App_minimal
    ```
3.  以下のコマンドを実行して、`Serena`サーバーを起動します。
    ```powershell
    uvx --from git+https://github.com/oraios/serena serena-mcp-server --context ide-assistant
    ```4.  サーバーのログ（`INFO: Uvicorn running...`など）が表示されたら、そのウィンドウは**作業がすべて終わるまで、そのまま開いたままにしておきます。**
    *   **（重要）このウィンドウは最小化しても構いませんが、絶対に閉じないでください。**車のエンジンのようなものです。

#### ステップ2: Serenaクライアントを起動して接続する (2つ目のPowerShell)

次に、`Serena`と対話するためのクライアントを起動します。

1.  **【新しい】PowerShell**をもう一つ開きます。（ステップ1のウィンドウとは別のウィンドウです）
2.  こちらも、同じプロジェクトフォルダに移動します。
    ```powershell
    cd C:\Users\mitsu\Desktop\X_Grid_App_minimal
    ```
3.  `serena`コマンドでクライアントを起動します。
    ```powershell
    serena
    ```
4.  `>`というプロンプトが表示され、対話できる状態になります。

#### ステップ3: プロジェクトを有効化する (クライアント内での作業)

最後に、これから作業するプロジェクトを`Serena`に伝えます。

1.  `serena`クライアントのプロンプトで、以下のコマンドを実行します。
    ```
    > @activate_project
    ```
2.  プロジェクト名を聞かれたら、 `.` (ドット) を入力してEnterを押します。
3.  `Project 'X_Grid_App_minimal' activated.` のようなメッセージが表示され、プロンプトの横にプロジェクト名が表示されたら準備完了です。

---

### まとめ：毎日の起動手順 (最短版)

1.  **ターミナル①:** プロジェクトフォルダに移動 → `uvx ...` でサーバーを起動 → **そのまま放置**
2.  **ターミナル②:** プロジェクトフォルダに移動 → `serena` でクライアントを起動
3.  **クライアント内:** `@activate_project` を実行

これだけです！

**ポイント:**
*   `@onboarding`は初回だけで、毎日の作業では不要です。
*   `Serena`は`.serena/memories`フォルダから前回の記憶を読み込むので、昨日の作業の続きから始められます。

この手順をメモしておけば、明日からいつでもスムーズに`Serena`とのコーディングを再開できます。
